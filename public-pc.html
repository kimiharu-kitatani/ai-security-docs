社外でパソコンを使う時のセキュリティ

カフェ・ホテル・コワーキングスペース 完全リスクガイド

対象：テレワーク・外出先で仕事をするエンジニア・ビジネスパーソン
目的：「自宅・会社以外」でPCを使う際の落とし穴をすべて把握する

CHAPTER 1｜物理的な脅威

👁️ ショルダーサーフィン（のぞき見）

最も多く、最もシンプルな攻撃です。機材不要・誰でもできます。

後ろや横に座った人が、画面やキーボード入力を直接目視で盗む

盗まれるもの：
  → パスワード（キーボードの打鍵を目で追う）
  → 画面に表示されているビジネス情報・顧客情報
  → ログイン後の画面（どのサービスを使っているか）
  → クレジットカード番号の入力シーン

実態データ

HPが実施した調査：視覚的ハッキングの試みが 88% 成功
ロンドン・NYオフィス街での実験：91%のケースで機密情報が盗み見られた

危険な場所ランキング

場所 危険度 理由
新幹線・飛行機 🔴 最高 後ろの席から完全に見える・長時間
カフェ 🔴 高い 密集・背後に人が通る
コワーキングスペース 🔴 高い 同じ空間で業種バラバラな他人がいる
図書館 🟡 中 比較的空いているが油断しがち
ホテルロビー 🟡 中 通行人が多い

対策

✅ プライバシーフィルター（覗き見防止フィルム）を貼る
   → 正面以外からは画面が黒く見える　費用：2,000〜5,000円

✅ 壁を背にして座る（後ろを取らせない）

✅ パスワード入力前に周囲を確認する習慣をつける

✅ 機密性の高い作業は公共の場でやらない

🔌 USB Rubber Ducky 攻撃（席を離れた「数秒」で完了）

スパイ映画のシーンがそのまま現実に存在します。

攻撃ツール：USB Rubber Ducky（市販品・約8,000円）
見た目：普通のUSBメモリと全く同じ
実態：接続した瞬間「キーボード」として認識される

攻撃の流れ

① あなたがトイレ・注文などで席を離れる（ロックし忘れでも5秒でOK）
      ↓
② 攻撃者がUSBを挿す（2〜3秒）
      ↓
③ PCは「キーボードが接続された」と判断
   ※ウイルス対策ソフトは反応しない（キーボードとして振る舞うため）
      ↓
④ 人間の1000倍の速さで事前プログラムされたコマンドを自動実行
      ↓
⑤ 10〜30秒でマルウェア設置・データ抜き取り完了
      ↓
⑥ USBを抜いて立ち去る → 痕跡ほぼなし

何ができるか

・バックドア（遠隔操作の入口）のインストール
・パスワードマネージャーから認証情報を抜き取る
・SSHキー・APIキーのコピー
・カメラ・マイクを密かに有効化
・次回起動時に自動でマルウェアを動かす仕込み

ロック画面があれば基本的に防げます。
「ちょっとだけ席を外す」が最も危険です。攻撃は数十秒で完了します。

💾 BadUSB 攻撃（見た目で識別できない）

普通のUSBメモリ・マウス・充電ケーブルに見えるが、
ファームウェアレベルでマルウェアが仕込まれている

怖い点：
  ❌ ウイルス対策ソフトでは検出できない（ファームウェアの中にあるため）
  ❌ 見た目で識別できない
  ❌ 接続するだけで感染する

実際の事例

2010年 Stuxnet事件： イランの核施設はインターネットに繋がっていなかったが、
工作員がUSBを持ち込んだことで感染。ウラン濃縮遠心分離機を物理的に破壊した。

USB ドロップ攻撃： 駐車場に放置された 297本のUSBのうち
45% が拾われてPCに挿された（IEEE研究）

対策

✅ 拾ったUSBは絶対に挿さない
✅ 見知らぬ人から受け取ったUSBは使わない
✅ USBポートロック（物理的に塞ぐキャップ）を使う
✅ BIOS設定でUSBブートを無効化する

🏨 Evil Maid 攻撃（ホテルの「清掃員」）

「部屋にPCを置いて外出する」のが危険です。

名前の由来：
  ホテルに泊まっている間、部屋の清掃員（悪意あるスタッフ）が
  放置されたPCを細工する、という攻撃シナリオから

手口：
  1. ホテルの部屋・会議室にPCを置いたまま外出
  2. 攻撃者がUSBから特殊ツールを起動
  3. ディスク暗号化キーを抜き取る（特定条件ではBitLockerでも可能）
  4. または小型キーロガーをキーボードとPC本体の間に物理的に挿す

現実的な標的： ビジネス出張者・外交官・記者・エグゼクティブ
一般の観光客よりも「価値ある情報を持つ人」が狙われます。

対策

✅ ホテルを出るときはPCをスリープでなく「電源OFF」にする
   （スリープ状態はメモリに暗号化キーが残るため危険）
✅ 重要なPCはホテルに置いて外出しない
✅ BitLocker/FileVault + TPMチップを有効にする
✅ Secure Bootを有効にする

🔋 ジュースジャッキング（空港の充電ポート）

FBIが公式警告。ただし現実の被害は少ない。

改ざんされたUSB充電ポートに接続すると：
  → 充電されながら同時にデータ転送が行われる
  → マルウェアがインストールされる

現在の見方：
  → iPhoneもAndroidも「このデバイスを信頼しますか？」と確認する
  → 2026年6月時点でも、スマートフォン一般利用者の大規模な実被害は確認されていない
  → ただし FBI・FCC・TSA は公衆USB充電への注意喚起を継続
  → 2025年には研究上の新しいUSB攻撃手法も報告されており、低頻度だが対策推奨

対策

✅ USB充電ポートではなく、コンセント（AC）から充電する
✅ USB Data Blocker（「USBコンドーム」）を使う
   → 充電のみ・データ通信を物理的に遮断　費用：数百〜千円

🔒 物理的な盗難

最も単純で最も多い「攻撃」

リスク：
  → PCそのものが盗まれてデータが流出
  → ディスクが抜かれて別PCで読まれる（暗号化なしの場合）

対策

✅ Kensington ロック（PCをテーブルに固定するワイヤー）
✅ フルディスク暗号化（BitLocker / FileVault）を有効にする
   → ディスクを抜かれても読めない
✅ 席を離れるときはカバンの中にしまう

CHAPTER 2｜Wi-Fi ネットワークの脅威

✅ 外出先Wi-Fi 日常チェックリスト

接続前：
  □ 店員・施設案内・公式掲示で「正しいWi-Fi名（SSID）」を確認した
  □ 似た名前のWi-Fiが複数ないか確認した（例：Airport_WiFi / Airport_Free_WiFi）
  □ パスワードなし・利用規約だけのWi-Fiで重要作業をしない
  □ 迷ったらフリーWi-Fiではなくスマホのテザリングを使う

接続時：
  □ 「自動的に接続する」をOFFにしてから接続した
  □ ログイン画面で、SNS・Google・Microsoft等のID/パスワードを入力しない
  □ 接続後すぐVPNをONにし、VPN接続を確認してから作業する

利用後：
  □ 使い終わったWi-Fiを「このネットワークを削除」した
  □ 空港・ホテル・カフェのWi-Fiを保存したままにしていない
  □ 機密作業・送金・管理画面ログインは可能なら社内回線かテザリングで行う

📡 Evil Twin 攻撃（偽の同名Wi-Fi）

最も成功率の高いWi-Fi攻撃です。

本物：  "Starbucks_WiFi"  （本物のWi-Fi）
偽物：  "Starbucks_WiFi"  （攻撃者が設置した全く同名のWi-Fi）

あなたのPCはどちらが本物か区別できない

攻撃の仕組み

① 攻撃者がWiFi Pineapple（攻撃専用デバイス・約25,000円）を
  カバンに入れてカフェに入店
      ↓
② 本物より電波を強くして自動接続させる
      ↓
③ 接続したすべての通信が攻撃者のデバイスを経由する
      ↓
④ ログイン情報・メール・ファイルがすべて傍受可能
      ↓
⑤ HTTPS通信もSSL Strippingで強制的にHTTPにダウングレードできる

実際の被害事例

ロンドンのカフェ・ホテルで調査した結果、
80,000件以上のプライベートデータが違法傍受されていたことが判明（実際の事件）

対策

✅ 正しいWi-Fi名（SSID）を確認してから接続する
   → 店員・受付・公式掲示・公式アプリで確認する
   → 「Free」「Secure」「5G」などが付いた似た名前に注意

✅ VPNを使う
   → Evil Twin経由でも、通信は暗号化されるため傍受できない
   → ただし偽ログイン画面への入力ミスはVPNでは防げない

✅ Wi-Fiの自動接続をOFFにする
   → Windowsの設定 → ネットワーク → 「自動的に接続する」をOFF
   → Macの設定 → Wi-Fi → 「ネットワークに自動接続」をOFF

✅ 使い終わったWi-Fiはすぐ「このネットワークを削除」する

✅ 重要作業はスマホのテザリングを優先する
   → 空港・ホテル・カフェの共有Wi-Fiより安全

✅ HTTPSのサイトのみ使う（ブラウザのアドレスバーに🔒マークがある）

🍍 WiFi Pineapple（専用攻撃デバイス）

Hak5社が販売する「ペネトレーションテスト用」ツール
値段：約200ドル（誰でも買える）
大きさ：スマートフォン程度

できること：
  ・Evil Twin攻撃の完全自動化
  ・接続した全デバイスの通信をリアルタイム記録
  ・HTTPSを偽装した偽ログインページの表示
  ・通信内容の改ざん（マルウェア注入）

なぜ「Airport_WiFi」に自動接続してしまうのか

あなたのPC・スマホが過去に「Airport_WiFi」に接続した記憶を持っている
      ↓
端末は同じ名前のWi-Fiを見つけると「前に使ったネットワーク」と判断する
      ↓
WiFi Pineappleが同じSSIDを発信していると、自動接続することがある
      ↓
攻撃者の機器を経由して通信してしまう

PineappleがSSIDを知る主なパターン：
  ① 周辺にある本物のWi-Fi名を見て、その場で同じSSIDをコピーする
  ② 端末が「保存済みWi-Fiを探す信号」を出している場合、そのSSIDを拾う
  ③ よくある名前（Airport_WiFi、Free_WiFi、Hotel_WiFi等）をあらかじめ用意する

注意：
  最近のiPhone・Android・Windows・Macは、過去SSIDをむやみに出さない改善が進んでいる
  それでも「自動接続ON」「パスワードなしWi-Fi」「保存済みWi-Fiが多い」ほど危険

WiFi Pineappleへの対策

✅ 公衆Wi-Fiは原則「自動接続OFF」にする
✅ 旅行後・出張後に空港/ホテル/カフェのWi-Fiを削除する
✅ 同じ名前のWi-Fiが複数ある場合は接続しない
✅ 正しいSSIDを施設スタッフ・公式表示で確認する
✅ 機密作業はテザリングか信頼済み回線を使う
✅ VPNを常時ONにする。ただし偽ログイン画面には情報を入れない

📦 パケットスニッフィング

同じWi-Fiに繋がっているだけで、
Wireshark（無料ソフト）を使えば他人の通信が見える

見えるもの：
  → HTTPサイトのIDとパスワード（平文で丸見え）
  → Cookie（ログイン状態） → セッションハイジャック
  → どのサイトにアクセスしているか（ドメイン名）

見えないもの：
  → HTTPSの通信内容（暗号化されているため）

🎣 偽キャプティブポータル（偽の「Wi-Fi接続確認画面」）

カフェのWi-FiにつなぐとSometimes「同意して接続」画面が出る
→ あの画面が偽物にすり替えられることがある

攻撃者が作った偽の確認画面でできること：
  ・メールアドレス・パスワードを入力させて盗む
  ・「ドライバの更新が必要です」とマルウェアをダウンロードさせる
  ・本物そっくりのGoogleログイン画面でGoogle認証情報を盗む

見分け方・対策

✅ URLを確認する（httpsになっているか、本物のドメインか）
✅ VPNを先に起動してから接続する
✅ 「ソフトウェアの更新」を求める画面は無視する（本物のWi-Fiは求めない）

CHAPTER 3｜Bluetooth の脅威

Bluesnarfing（ブルースナーフィング）

範囲：約10m（Bluetooth 5.0では理論上100m以上も可能）
条件：Bluetoothが「発見可能」モードになっている

盗まれるもの：
  → 連絡先一覧
  → メール・カレンダー
  → ファイル
  → 通話履歴

Bluebugging（ブルーバッギング）

Bluesnarfingより高度な攻撃。デバイスを遠隔操作できる

できること：
  → 電話をかける（勝手に）
  → SMSを勝手に送信
  → マイクを有効化して盗聴
  → デバイス内のデータにアクセス

対策

✅ 使わないときは Bluetooth を OFF にする（これだけで99%防げる）
✅ 「発見可能」モードを常にOFFにする
✅ 知らない相手からのペアリング要求を承認しない

CHAPTER 4｜「映画レベル」の高度な脅威

これらは一般人への攻撃ではほぼ発生しません。
外交官・エグゼクティブ・機密情報を扱う方 は参考にしてください。

📡 TEMPEST攻撃（電磁波盗聴）

PCのモニターが発する電磁波を、
数メートル〜数百メートル外から専用アンテナで受信し、
画面内容を再現する技術

TEMPEST = TEMPorary Emanation and Spurious Transmission（NATOの機密文書用語）

現実：
  → 政府・軍の機密施設では本当に「電波遮蔽室（ファラデーケージ）」で対策
  → 一般の攻撃者には機材・技術の敷居が非常に高い
  → 研究者による実証実験は成功している（論文あり）

現実的な脅威度：スパイ活動・国家レベルの標的型攻撃

🎹 音響キーボード解析

キーボードの打鍵音を録音し、AIで解析してキー入力内容を再現する

2023年 英国の研究者の実験：
  → スマートフォンのマイク（隣の席に置くだけ）で録音
  → AIで解析した結果、95%の精度でキー入力を再現
  → ZoomやSkypeの通話音声からも 93%の精度で解析できた

現実的な脅威度：研究レベルでは実証済み。実際の攻撃への使用例は少ない。

🌡️ サーマルカメラ攻撃（熱残留解析）

パスワード入力後、キーボードのキーには体温が残る
赤外線カメラで撮影すると、どのキーを押したか分かる

2022年 グラスゴー大学の実験：
  → 入力後 30秒以内なら 86%の精度でパスワードを特定
  → 入力後 60秒以内でも 80%以上

対策：パスワード入力後、手のひらをキーボード全体に置いて熱を均一にする

🪟 リフレクション攻撃（反射での盗み見）

カフェの窓ガラス・眼鏡のレンズ・コーヒーカップの表面・
向かいの人の瞳などへの反射を望遠レンズで撮影し、
画面内容を読み取る

対策：プライバシーフィルター＋壁を背にして座る

CHAPTER 5｜ソーシャルエンジニアリング

🎭 偽のITサポート

「Wi-Fiの設定をお手伝いします」「ソフトウェアの更新を代わりに」
と近づいてくる

→ 実はPCを操作してマルウェアを仕込む
→ または言葉でパスワードを聞き出す

🧲 「荷物を見ていてください」手口

見知らぬ人が「少し席を外すので荷物を見ていてください」と頼む
→ あなたが目を離した隙に、共犯者があなたのPCを操作する
→ または本人が戻ってきたと見せかけてUSBを抜く

💿 USBドロップ攻撃

「CONFIDENTIAL」「給与一覧2026」等と書いたUSBを
駐車場・カフェ・エレベーター前に置いておく
→ 好奇心で拾った人が自分のPCに挿す
→ 自動的にマルウェアが動く

IEEEの実験：放置したUSBの 45% が拾われてPCに挿された

📌 場所別 具体的なアクションガイド

カフェで仕事するとき

入店時：
  □ 壁を背にした席を選ぶ
  □ プライバシーフィルターを貼ったPCを使う
  □ 正しいWi-Fi名を店員・公式掲示で確認
  □ 似た名前のWi-Fiが複数ある場合はテザリングに切り替える
  □ VPNに接続してから作業開始

作業中：
  □ パスワード・APIキーの入力前に後ろを確認
  □ ネットバンキング・クレジットカード操作は避ける
  □ Bluetoothは不要なら OFF

離席時：
  □ 必ずスクリーンロック（Windows: Win+L / Mac: Cmd+Ctrl+Q）
  □ PCをカバンの中にしまってから離席
  □ 「ちょっとだけ」でもロックする習慣を徹底

退席時：
  □ 接続したWi-Fiを「削除」して自動接続をリセット

ホテルに泊まるとき

□ PCをホテルの部屋に置いて外出しない（可能な限り）
□ 外出する場合はPCを「電源OFF」にする（スリープ NG）
□ ホテルのWi-FiはVPN経由のみ使う
□ フロントが見えるカフェエリアでの作業もリスクあり
□ 「部屋のUSBポート」への充電は避けACアダプタを使う

新幹線・飛行機の中

□ プライバシーフィルターは必須（隣・後ろから丸見え）
□ 機内Wi-FiはVPN経由のみ使う（傍受リスク高）
□ 席を離れる際はPCをスリープでなくロック
□ PCはひざの上に乗せてカバンには戻さなくていいが画面は閉じる

🛡️ 今すぐ実施すべき対策セット

無料・設定だけでできること

✅ スクリーンロックを「30秒〜1分後」に自動設定
   Windows: 設定 → 電源とスリープ → スクリーンセーバー設定
   Mac: システム設定 → スクリーンセーバー → ロックまでの時間

✅ フルディスク暗号化を有効化
   Windows: BitLocker（Proエディション）
   Mac: FileVault（システム設定 → プライバシーとセキュリティ）

✅ Wi-Fiの「自動接続」をOFF
   使い終わったWi-Fiは「このネットワークを削除」する

✅ 公衆Wi-Fiの正しいSSIDを確認する習慣をつける
   店員・受付・公式掲示で確認し、似た名前には接続しない

✅ Bluetoothを使わないときはOFF

✅ 「発見可能」モードをOFF（Bluetoothの設定）

数千円でできること

✅ プライバシーフィルター（覗き見防止フィルム）
   → Amazonで購入可能・2,000〜5,000円

✅ Kensingtonロック（盗難防止ワイヤー）
   → カバンを開けてPCをしまえない混雑した場所で有効
   → 2,000〜5,000円

✅ USB Data Blocker（USBコンドーム）
   → 充電のみ・データ通信を物理的に遮断
   → 数百〜千円

月額数百〜千円でできること

✅ 有料VPN（ProtonVPN / Mullvad VPN / NordVPN等）
   → フリーWi-Fiでの全通信を暗号化
   → パケットスニッフィングを大きく軽減
   → 偽ログイン画面や誤接続そのものは防げないためSSID確認も必要

✅ パスワードマネージャー（1Password / Bitwarden等）
   → 画面を見られてもパスワードは表示されない
   → キーボードを見られてもオートフィルで防止

📌 このCHAPTERの結論

社外での最大のリスクは「物理的な脅威」と「フリーWi-Fi」です。
高度な電磁波攻撃よりも、「のぞき見」と「偽Wi-Fi」の方がずっと現実的な脅威です。

状況 最重要対策
カフェ・コワーキング プライバシーフィルター ＋ VPN ＋ 離席時ロック
ホテル出張 外出時は電源OFF ＋ 部屋にPC置かない
新幹線・飛行機 プライバシーフィルター（必須）
フリーWi-Fi全般 VPNなしで機密作業・ネットバンキングは絶対しない
USB関連 拾ったUSBは挿さない・離席時は必ずロック

一言まとめ： 「壁を背に座る」「VPNを使う」「離席時は必ずロック」
この3つを習慣にするだけで、日常的なリスクの90%が防げます。
